CREWTUI DOCUMENTATION
=====================

Version 1.0
Last updated: 2026-04-09


OVERVIEW
--------

CrewTUI is a config-driven terminal interface for managing multi-agent
AI crews. You define agents with roles, goals, and tools. They work
together on missions you assign — from the terminal, Telegram, or on
a schedule.

No code changes needed. Everything is configured through the setup
wizard or JSON config files.


INSTALLATION
------------

Prerequisites:
  - Python 3.12 or higher
  - uv package manager (recommended) or pip
  - At least one AI provider API key
  - Optional: LM Studio or Ollama for local models
  - Optional: Telegram bot token for remote control

Steps:
  1. Clone the repository
     git clone https://github.com/cliftonwknox/crewtui.git
     cd crewtui

  2. Install dependencies
     uv sync
     uv pip install -e .

  3. Run the setup wizard
     uv run crewtui setup

  4. Launch the TUI
     uv run crewtui

The setup wizard creates:
  - project_config.json   Your agents, tools, and routing rules
  - .env                  API keys (stored securely in your work directory)
  - heartbeat_config.json Heartbeat interval and auto-start settings
  - Desktop shortcut      Optional, for your application menu


CLI COMMANDS
------------

  crewtui              Launch the TUI (default)
  crewtui setup        Run the first-time setup wizard
  crewtui models       Manage model presets (list, add, remove, test)
  crewtui telegram     Configure Telegram notifications
  crewtui daemon on    Start the background daemon
  crewtui daemon off   Stop the background daemon
  crewtui daemon status Check daemon status


TUI COMMANDS
------------

Crew:
  /crew <mission>            Run a full crew mission
  /crew @agent <mission>     Run a mission with a specific agent

Queue:
  /queue add <description>   Add a task to the queue
  /queue add --crew <desc>   Queue a full crew run
  /queue add @agent <desc>   Queue task for specific agent
  /queue list                Show all tasks
  /queue cancel <id>         Cancel a pending task
  /queue clear               Remove completed/failed tasks
  /queue remove <id>         Remove a specific task

Heartbeat:
  /heartbeat on              Start the heartbeat engine
  /heartbeat off             Stop the heartbeat engine
  /heartbeat status          Show heartbeat status
  /heartbeat interval <N>    Set check interval in seconds

Cron:
  /cron add                  Start the cron job wizard
  /cron list                 Show all scheduled jobs
  /cron remove <id>          Delete a cron job
  /cron on <id>              Enable a cron job
  /cron off <id>             Disable a cron job
  /cron run <id>             Trigger a cron job immediately
  /cron approve <id>         Approve an agent-proposed job
  /cron reject <id>          Reject an agent-proposed job

  Quick add (skip wizard):
  /cron add Name | schedule | mission description
  /cron add Name | schedule | @agent mission description

  Schedule formats:
    daily 08:00              Every day at 8:00 AM
    weekly mon 9:00          Every Monday at 9:00 AM
    every 6h                 Every 6 hours
    hourly                   Every hour
    monthly 1 09:00          First of every month at 9:00 AM

Skills:
  /skills                    Show the Skills tab
  /skills install <tool>     Enable a tool
  /skills assign <tool> <agent>    Give tool to an agent
  /skills unassign <tool> <agent>  Remove tool from agent
  /skills new                Create a custom skill template
  /skills refresh            Rescan skills directory

Files:
  /view <number>             View a file by its list number
  /view <name>               View a file by partial name match
  /delete <file>             Delete an output file
  /delete all                Delete all output files
  /purge                     Delete all output files (shortcut)
  /open                      Open the output directory

Configuration:
  /config <agent> <preset>   Change an agent's model preset
  /presets                   List available model presets

Memory:
  /memory                    Show agent memory
  /remember <text>           Add a memory entry
  /forget <text>             Remove a memory entry

Telegram:
  /telegram show             Show Telegram configuration
  /telegram on               Enable notifications
  /telegram off              Disable notifications
  /telegram test             Send a test message
  /telegram listen           Start the Telegram listener

Daemon:
  /daemon on                 Start the background daemon
  /daemon off                Stop the daemon
  /daemon status             Check if daemon is running

System:
  /restart                   Restart TUI and daemon (reloads all code)
  /exit                      Close TUI (daemon keeps running)
  /help                      Show command help
  /clear                     Clear the current panel
  /copy                      Copy panel content to clipboard
  /status                    Show agent status summary
  /history                   Show crew run history

TELEGRAM COMMANDS
-----------------

Send these to your CrewTUI Telegram bot:

  /crew <mission>        Queue a full crew run
  /task <description>    Queue a single-agent task
  /task @agent <desc>    Queue task for specific agent
  /status                Current crew and heartbeat status
  /history               Recent crew run history
  /queue                 Show task queue
  /agents                List configured agents
  /crons                 List scheduled cron jobs
  /approve <id>          Approve a pending cron job
  /reject <id>           Reject a pending cron job
  /runcron <id>          Trigger a cron job immediately
  /help                  Show command list

  Or just send any text message to queue it as a crew mission.


DAEMON
------

The daemon runs the heartbeat engine and Telegram listener in the
background. It starts automatically when you launch the TUI and
keeps running after you close it.

  - Telegram missions are processed even when the TUI is closed
  - Cron jobs fire on schedule
  - Queue tasks are picked up every heartbeat interval (default 60s)
  - Reports are saved to the output directory and sent via Telegram

The daemon stops when you run /daemon off or crewtui daemon off.
It restarts fresh when you use /restart in the TUI.


AGENTS AND MODELS
-----------------

Agents are defined in project_config.json. Each agent has:
  - id          Unique identifier (e.g., "researcher")
  - name        Display name (e.g., "Research Analyst")
  - role        What CrewAI sees (e.g., "Senior Research Analyst")
  - goal        The agent's objective
  - backstory   Personality and expertise description
  - preset      Which AI model to use
  - tools       List of tools the agent can use
  - color       Display color in the TUI

IMPORTANT: Do not use "manager" in an agent's id or role if the
agent has tools assigned. CrewAI treats "manager" agents as
hierarchical managers and blocks tool access. Use "coordinator",
"lead", or "director" instead.

MODELS TAB
----------

The Models tab lets you manage model presets and agents without
editing config files.

Left sidebar:
  - Model presets with API key status and which agents use them
  - Agents with their assigned preset and tool count
  - Click any item to load it into the form for editing

Model form:
  - New Model button to clear the form for a fresh entry
  - Load Existing Preset dropdown to copy a built-in template
  - Fields: Preset Name, Display Label, Model ID, Base URL,
    API Key Env Var, Provider Name, API Compatibility
  - API Compatibility: OpenAI (most providers) or Anthropic
  - Test button sends a test prompt to verify the connection
  - Save button stores the preset (max 10 custom presets)
  - Delete button removes custom presets (built-ins protected)

Agent form:
  - New Agent button to clear the form
  - Fields: Agent ID, Display Name, Role, Goal, Backstory,
    Model Preset, Tools, Color, Routing Keywords
  - Tools are comma-separated (e.g., ddg_search, crewai:FileReadTool)
  - Save button creates or updates the agent
  - Delete button removes the agent (cannot delete last agent)
  - Changes require /restart to take effect

Custom models:
  Any provider with an OpenAI-compatible or Anthropic-compatible
  API can be added. Fill in the provider name, model ID, API key
  environment variable, base URL, and select the API compatibility.
  For OAuth providers, resolve the token externally and paste it
  as an API key.


TOOLS
-----

Three tiers of tools are available:

1. Built-in tools (always available):
   ddg_search        DuckDuckGo web search (free, no API key)
   tavily_search     Tavily AI search (needs TAVILY_API_KEY)
   scrape_website    Scrape any URL
   cron_tool         Create and manage scheduled cron jobs

2. CrewAI ecosystem tools (install via /skills):
   FileReadTool, FileWriterTool, DirectoryReadTool,
   TXTSearchTool, and more from the crewai_tools package.

3. Custom skills:
   Drop a Python file in your project's skills/ directory.
   Must contain a class inheriting from crewai.tools.BaseTool.
   Use /skills refresh to pick up new skills.


CUSTOM SKILLS
-------------

Create a file in your project's skills/ directory:

  from crewai.tools import BaseTool

  class MyCustomTool(BaseTool):
      name: str = "My Custom Tool"
      description: str = "Describe what this tool does"

      def _run(self, query: str) -> str:
          # Your logic here
          return f"Result for: {query}"

Then run /skills refresh in the TUI. Assign it to agents with
/skills assign my_custom_tool agent_id.


CRON SCHEDULING
---------------

Cron jobs run missions on a schedule. The daemon checks for due
jobs on each heartbeat tick and queues them automatically.

There are three ways to create cron jobs:

1. TUI wizard: /cron add (interactive 6-step wizard)
2. Agent proposal: any agent with the cron_tool can create jobs
   (requires user approval via TUI or Telegram)
3. Quick add: /cron add Name | schedule | mission

Agent-proposed crons go to "pending approval" status. Approve them
with /cron approve <id> in the TUI or /approve <id> on Telegram.

Reports from cron jobs are saved to the output directory and sent
to Telegram. Use the report option in the wizard to control this.


REPORT CONTEXT
--------------

Every task and crew run automatically receives context about recent
reports. The 3 most recent report contents are injected into the
task prompt so agents can reference previous work without needing
file access tools.

Agents with FileReadTool and DirectoryReadTool can dig deeper into
older reports in the output directory.


FILE BROWSER
------------

The Files tab shows output files with filter tabs:

  Chronological    All files sorted by date (newest first)
  Recurring        Heartbeat/cron task outputs
  Reports          Full crew reports
  Decisions        Decision documents

Click a file in the list to view it. Use /view <number> to open
a file by its position in the list.


CONFIGURATION FILES
-------------------

All configuration files are stored in your project's work directory
(set during setup). They are not tracked by git.

  project_config.json      Agents, tools, routing rules
  .env                     API keys
  heartbeat_config.json    Heartbeat settings
  telegram_config.json     Telegram bot configuration
  cron_config.json         Scheduled job definitions
  model_presets.json       Custom model presets
  memory/                  Agent memory storage
  output/                  Generated reports and documents
  skills/                  Custom skill files

See .env.example and project_config.sample.json in the repository

TROUBLESHOOTING
---------------

Agent says "Manager agent should not have tools":
  Rename the agent. Do not use "manager" in the id or role field.
  Use "coordinator", "lead", or "director" instead.

Agent says it cannot access the internet:
  Make sure the agent has ddg_search or tavily_search in its tools
  list. Single-agent tasks now run through CrewAI with full tool
  access.

Telegram messages are cut off:
  Long messages are automatically split into numbered parts.
  If still truncating, restart the daemon to load the latest code.

Daemon won't stop:
  The daemon uses SIGKILL as a fallback if SIGTERM fails.
  Run: crewtui daemon off
  Or manually: kill -9 <pid> && rm crewtui_daemon.pid

Zombie TUI processes:
  CrewTUI auto-kills orphaned processes on startup. If you see
  old processes, just restart and they'll be cleaned up.

Reports not appearing in Files tab:
  The tab auto-refreshes every 5 seconds. Switch to another tab
  and back, or run /view 1 to force a refresh.
